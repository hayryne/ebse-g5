            1. summary: AsyncAdmin can not deal with non default meta replica
              description: In AsyncAdmin we support passing both encodedRegionName and regionName as a parameter so we have a getRegionInfo method to get the RegionInfo first. In general we will scan the meta table to get the region info, but for meta region itself, the problem is that, we only check for the first replica, so for the non default replicas, we will still go to the meta region and then get an UnknownRegionException. We should find a way to deal with this.
