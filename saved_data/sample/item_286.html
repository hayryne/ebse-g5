    1. summary: Log Collation - Memory leak when all hosts are down.
      message: Log Collation - Memory leak when all hosts are down. This patch is intended to address issue #2416. When all hosts are down, the log output is supposed to be written to an orphan file. The LogHost::orphan_write_and_try_delete() is supposed to consume a LogBuffer reference count, but it only does so in one branch of the if/then/else instead of both. This patch fixes this issue. (cherry picked from commit 3bf26d17e6bb8aed07de5681a9bdc75e59b72294)
