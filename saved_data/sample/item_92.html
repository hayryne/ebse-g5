            1. summary: User Guide: GFSH GC command should be documented to use only in non-production environments
              description: [DWisler] The GSFH GC command is very likely to cause members of GemFire to be kicked out due to being unresponsive, as this is a forced GC event and not always handled well by GC outside of normal GC processing. We need to document that this should only be used in non-production environments, and to not use it if it causes such issues. We are especially susceptible if multiple members run on the same vm/host and/or if the number of cpus is insufficient to handle the “load” needed by GC.
