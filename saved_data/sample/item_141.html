            11. I don't like the approach of commenting this line out. I think a lot of admins will infer from this that uncommenting the line will result in some useful behavior. A bigger problem is that hadoop-env.sh is sometimes included twice. For example, hadoop-daemon.sh includes hadoop-config.sh, which includes hadoop-env.sh. Then later hadoop-daemon.sh includes hadoop-env.sh explicitly. It seems to me that this is the actual root cause of the issue, and that it is likely going to cause other headaches in the future by people who don't suspect that it will be included twice.
